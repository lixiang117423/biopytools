# KMERIA Binary File Viewing Guide

## Overview

KMERIA count命令生成的.bin二进制文件可以通过`kdump`工具转换为文本格式进行查看。

The binary .bin files generated by KMERIA count command can be converted to text format using the `kdump` tool.

## Tool Location

```
/share/org/YZWL/yzwl_lixg/software/kmeria/bin/kmeria dump
```

Or via biopytools wrapper:
```bash
# 需要先设置环境变量|Need to set environment variables first
export LD_LIBRARY_PATH=/share/org/YZWL/yzwl_lixg/software/kmeria/lib:/share/org/YZWL/yzwl_lixg/miniforge3/envs/kmeriaenv/lib:$LD_LIBRARY_PATH
```

## Usage

### Basic Conversion

Convert entire binary file to text (output to screen):
```bash
export LD_LIBRARY_PATH=/share/org/YZWL/yzwl_lixg/software/kmeria/lib:/share/org/YZWL/yzwl_lixg/miniforge3/envs/kmeriaenv/lib:$LD_LIBRARY_PATH
/share/org/YZWL/yzwl_lixg/software/kmeria/bin/kmeria dump input_sample_k31.bin
```

Convert to file:
```bash
/share/org/YZWL/yzwl_lixg/software/kmeria/bin/kmeria dump input_sample_k31.bin -o output.txt
```

### Output Formats

The `-O` option controls the output format:

| Format | Description | Example Output |
|--------|-------------|----------------|
| `ALL_COUNTS` | k-mer followed by all sample counts (default) | `ACGT\t10,20,15,8` |
| `SUM` | k-mer followed by sum of counts | `ACGT\t53` |
| `DIFFERENCE` | k-mer followed by (cnt1 - cnt2 - ...) | `ACGT\t-33` |
| `MIN` | k-mer followed by minimum count | `ACGT\t8` |
| `MAX` | k-mer followed by maximum count | `ACGT\t20` |
| `N` | k-mer followed by Nth sample count (1-based) | `ACGT\t15` |

Examples:
```bash
# 只显示每个k-mer的总count数|Show only sum of counts for each k-mer
kmeria dump input.bin -O SUM -o output_sum.txt

# 显示每个k-mer的最小count|Show minimum count for each k-mer
kmeria dump input.bin -O MIN -o output_min.txt

# 显示第3个样本的count|Show count from 3rd sample
kmeria dump input.bin -O 3 -o output_sample3.txt

# 查看前20行|View first 20 lines
kmeria dump input.bin -O SUM | head -20
```

### Filtering Options

Filter k-mers based on count thresholds:

| Option | Description | Example |
|--------|-------------|---------|
| `-c INT[,...]` | Minimum count threshold | `-c 5` (count >= 5 for all samples) |
| `-C INT[,...]` | Maximum count threshold | `-C 100` (count <= 100 for all samples) |
| `-n INT` | Minimum samples that must pass filters | `-n 3` (present in at least 3 samples) |
| `-N INT` | Maximum samples that may pass filters | `-N 5` (present in at most 5 samples) |

Examples:
```bash
# 只保留count >= 5的k-mers|Keep only k-mers with count >= 5
kmeria dump input.bin -c 5 -o filtered.txt

# 只保留count在5-100之间的k-mers|Keep only k-mers with count between 5-100
kmeria dump input.bin -c 5 -C 100 -o filtered.txt

# 只保留在至少3个样本中都存在的k-mers（count >= 1）|Keep k-mers present in at least 3 samples
kmeria dump input.bin -n 3 -o filtered.txt

# 复杂过滤：样本1 count>=10，样本2 count>=20，至少在2个样本中满足|Complex filtering
kmeria dump input.bin -c 10,20 -n 2 -o filtered.txt
```

### Practical Examples

1. **Quick inspection of k-mer counts** (快速查看k-mer计数):
```bash
kmeria dump sample1_k31.bin -O SUM | head -20
```

2. **Export high-abundance k-mers** (导出高丰度k-mers):
```bash
kmeria dump sample1_k31.bin -c 100 -O SUM -o high_abundance_kmers.txt
```

3. **Compare k-mers across specific samples** (比较特定样本间的k-mers):
```bash
kmeria dump matrix_k31.bin -O 1,2 -o samples_comparison.txt
```

4. **Find common k-mers** (查找共有k-mers):
```bash
kmeria dump matrix_k31.bin -c 1 -n 10 -o common_kmers.txt
# 至少在10个样本中count>=1的k-mers|K-mers with count>=1 in at least 10 samples
```

5. **Extract unique k-mers for one sample** (提取单个样本特有k-mers):
```bash
kmeria dump matrix_k31.bin -O DIFFERENCE -o unique_kmers.txt
# count1 - count2 - count3 - ... (正数表示样本1特有|positive means sample1-specific)
```

## Notes

1. **Performance**: 对于大的.bin文件，转换可能需要较长时间和磁盘空间|For large .bin files, conversion may take significant time and disk space
2. **Memory**: kdump工具内存占用较小|kdump tool has low memory usage
3. **Output size**: 文本文件通常比二进制文件大很多|Text files are typically much larger than binary files
4. **Compression**: 可以使用gzip压缩输出文件|Can use gzip to compress output files:
   ```bash
   kmeria dump input.bin -o - | gzip > output.txt.gz
   ```

## Integration with biopytools

When using the biopytools kmeria module, .bin files are generated in:
```
01_kmer_counts/      # Individual sample k-mer counts
02_kmer_matrices/    # Combined k-mer matrices
03_filtered_matrices/ # Filtered k-mer matrices
```

To view these files:
```bash
# Single sample k-mer counts|单个样本k-mer计数
export LD_LIBRARY_PATH=/share/org/YZWL/yzwl_lixg/software/kmeria/lib:/share/org/YZWL/yzwl_lixg/miniforge3/envs/kmeriaenv/lib:$LD_LIBRARY_PATH
kmeria dump 01_kmer_counts/sample1_k31.bin -O SUM -o sample1_counts.txt

# Population k-mer matrix|群体k-mer矩阵
kmeria dump 02_kmer_matrices/kmer_matrix.kmers.bin -O SUM | head -100
```
